This is an HTML file in /directory-listing-demo/some-dir

When you click on some-dir/ in directory listing, it sends a request to the server for the URI /directory-listing-demo/some-dir. The server first checks if the URI is a file or a directory. If it is a file. It attempts to respond with the contents of the files. If it is a directory, it looks for index files in that directory. If no index files are found, and autoindex is enabled in the configuration file, then it responds with directory listing for that URI (its HTML is created dynamically by the C++ code!). If autoindex is off, then error 404 is returned.

In the present case, some-dir/ is a directory, which has a file index.html with the contents that you are currently reading.
